We're encountering an issue with the enquiry form.

  • source

Issue 1 - Invalid Regular Expression

When filling out your form, the browser throws the following error while submitting the form: 

Pattern attribute value [0-9()#&+*-=.]+ is not a valid regular expression

Invalid regular expression:
/[0-9()#&+*-=.]+/v:
Invalid character in character class

However, when we fill out our form using the same backend API, the data is submitted successfully without any errors.

Please compare your form with our working form and identify what is different. It appears there is an issue with one of the HTML validation attributes (most likely the pattern attribute on the Phone Number or Postcode field), rather than the backend.

Issue 2 - Chrome DevTools Warning

Additionally, Chrome DevTools is reporting another issue:

A form field element should have an id or name attribute.

Some form elements either have a missing or empty id/name attribute, which may prevent browser autofill and can affect form functionality.

Please ensure that:

  • Every form field has a valid name attribute.
  • No name attribute is blank or missing.
  • Each form field also has a unique id attribute (recommended for accessibility and browser autofill).
  • The name attributes exactly match the backend field names listed above.
